# J.P. Morgan Asset Management: Guide to the Markets

## Official Reference

- Asia PDF:
  `https://cdn.jpmorganfunds.com/content/dam/jpm-am-aem/global/en/insights/market-insights/guide-to-the-markets/guide-to-the-markets-asia.pdf`

## When To Read This File

- The user explicitly mentions `Guide to the Markets`
- The user wants a J.P. Morgan Asset Management PDF style rather than the
  public website style
- The page is a chart-heavy research delivery artifact, handbook, scorecard,
  or exported single-page report

## Style Summary

- Treat this as a handbook-style research visual system, not a marketing site
- Prefer very light gray or warm-gray page backgrounds instead of pure white
- Use black or near-black titles, medium-gray secondary text, and thin grid
  lines
- Avoid decorative shadows and ornamental web UI chrome
- Use flat panels, section strips, color-coded category chips, and compact
  institutional typography
- Let tables and charts dominate the page; reduce hero sections and slogans

## Chart Palette

- Primary highlight: bright cyan / cyan-blue
- Base comparison: graphite or dark gray
- Supporting categories: deep blue, orange, purple, deep green, grass green
- Grid lines: soft visible gray
- Heat or rank encoding: functional blue/green/orange/purple tints

### Practical Defaults

- `#11A7D9` primary cyan-blue
- `#0D5CAB` deep blue
- `#666A70` graphite gray
- `#F08A00` orange
- `#6F42C1` purple
- `#0A6B3F` deep green
- `#6C9F2A` grass green

## Composition Cues

- Use stronger table headers and disciplined row dividers
- Use category color bars or chips to separate sections
- Keep components square or only slightly rounded
- Make the page feel exportable and review-ready
- Keep motion subtle or omit it entirely

## Do Not Overdo

- Do not mix copper-heavy website accents into the chart system unless the user
  explicitly wants the public-site brand feel
- Do not turn this into a colorful SaaS dashboard
- Do not use gradients, glassmorphism, glowing cards, or soft floating panels

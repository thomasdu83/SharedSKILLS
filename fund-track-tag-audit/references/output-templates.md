# Output Templates

报告采用**混合格式**：

- **表格章节**（Markdown 表格）：「基金记录（zmdata）」和「字段审核」
- **列表章节**（`- ` 格式）：其余四章（结论摘要 / 策略画像 / 关键证据 / 口径说明）

---

## Table Cell Formatting Rules（MANDATORY）

表格单元格内如需换行，**必须且仅能使用标准 HTML 标签 `<br>`**，严禁使用物理回车。错误示例会导致 Markdown 表格语法断裂：

```
<!-- ✅ 正确 -->
| `raceName` | 空 | **需补充**<br>(建议: 期货主观) | 依据... |

<!-- ❌ 错误 -->
| `raceName` | 空 | **需补充**
(建议: 期货主观) | 依据... |
```

已找到历史审核报告。

- 审核时间：YYYY-MM-DD HH:MM:SS
- 报告路径：<absolute path>

如需基于当前 zmdata 和最新材料重新审核，请说「重新审核<基金名称>」。

## Multi-Fund Workflow

1. 逐基金解析、逐基金查历史报告。
2. 已存在历史且未要求刷新的，复用。
3. 用户要求刷新某基金，仅重跑该基金。
4. 所有单基金报告自动保存。
5. 在对话中显示一篇综合报告，默认不落盘。

## Single Fund

六段固定结构，不增不减。格式约定：

| 章节 | 格式 | 备注 |
|---|---|---|
| 结论摘要 | 列表 | 分层：一致项 → 异常项 → 独立前置条件句 |
| 基金记录（zmdata） | 表格（两列） | 字段-值映射，含主审/参考/同系列三层 |
| 字段审核 | 表格（四列） | 字段 / 当前值 / 结论 / 依据与说明 |
| 策略画像 | 列表 | 收益来源用二级列表按子策略拆行 |
| 关键证据 | 列表 | 3-6 条；多份且有时序/逻辑演进时末尾加 📌 证据链小结 |
| 口径说明 | 列表 | — |

```markdown
## <fund_name> 策略标签审核

### 结论摘要

<第一段，2–3 句：一致项 + 异常项（缺失/冲突），直接给出建议值或挂空决定。>
<第二段，1–2 句：不确定性来源 + 必须人工确认的事项。如果 raceName 已给出明确建议，写明"建议值为 XXX，需人工二次确认与同系列产品的策略一致性"；如果 fundType 挂空，写明"待人工确认实际净值波动后在波动序列中定档"。>

**证据强度**：**<高|中|低>**
**依据**：<一句话说明证据来源和冲突情况。>

---

### 基金记录（zmdata）

**主审记录**：<名称>（<代码> / <fund_id> / <记录类型>）

| 字段 | 值 |
|---|---|
| `fundID` / `fundCode` | <fund_id> / <fund_code> |
| `fundManageCompany` | <管理人全称> |
| `strategyType` | **<值或「空」>** |
| `raceName` | **<值或「空」>** |
| `fundType` | **<值或「空」>** |
| `benchID` (benchName) | <bench_id> (<bench_name>) |

**投后子基金参考记录**

- <代码>（<名称> / <fund_id> / <记录类型>）：`strategyType=<值>` / `raceName=<值或「空」>` / `fundType=<值或「空」>`

**同系列尽调参考记录**（如适用）

- <代码>（<名称> / <fund_id> / 尽调）：`strategyType=<值>` / `raceName=<值或「空」>` / `fundType=<值或「空」>`

**多命中说明**：<主审选择逻辑与参考记录用途。如无独立尽调记录，应明确说明依赖同系列推断。>

---

### 字段审核

| 字段 | 当前值 | 结论 | 依据与说明 |
|---|---|---|---|
| `strategyType` | <值> | **<一致\|需补充\|需修正\|需复核补充\|人工复核>** | <1–3 句话。引用证据来源和词汇表校验结果。> |
| `raceName` | <值> | **<结论>**<br><br>*(建议: <唯一候选值>)* | <1–2 句解释。如建议值与参考记录不一致，简要说明原因。如有策略一致性风险，用 ⚠️ 标注需要人工确认的具体问题。> |
| `fundType` | <值> | **需复核补充**<br><br>*(待人工确认)* | <解释：①为什么不能直接给候选值（同系列推断、基准/杠杆不一致、无直接证据）；②波动层级定档的前置条件（如"需结合投后实际净值波动率在 CTA 波动序列中定档"）。仅当目标产品有直接 P1 证据且波动层级可从材料合理推断时，才可给出具体候选值。> |

结论词：**一致** / **需补充** / **需修正** / **需复核补充** / **人工复核**。
（如有投后子基金字段需补充，追加行。）

### 策略画像

- **一级赛道**：<L1>
- **细分赛道**：<L2>
- **策略属性**：<主观/量化/混合 / 具体描述，1-2 行>
- **收益来源**：
  - *<子策略/来源 1>*：<一句话核心逻辑>
  - *<子策略/来源 2>*：<一句话核心逻辑>
  - *<子策略/来源 3>*：<一句话核心逻辑>
- **投资范围**：<品种/板块。容量相关数据附后。>
- **风险画像**：<波动层级 / 关键风控参数。止损、卡玛比等具体数字可附后。>
- **标签映射**：`strategyType=<值>` / `raceName=<值>` / `fundType=<值>`

---

### 关键证据

**P1**　<来源> / <文件名>
<一句摘要，控制在 2 行以内。>

**P1**　<来源> / <文件名>
<一句摘要。>

**P2**　zmdata 当前字段 / <查询时间或记录 ID>
<一句摘要，仅用于说明当前结构化字段基线，不作为 P1 尽调证据。>

（3-6 条，按优先级排列。优先级只允许 P1/P2。P1 为尽调证据；P2 仅用于当前 zmdata 字段基线。营销材料标注「营销材料，P1 中偏弱」。）

📌 **证据链小结**：<1-2 句话，按时间线或逻辑线串联关键证据，点明策略生命周期状态或边际变化。仅当证据在时间/策略线上有演进关系时必写；单份证据时省略。>

---

### 口径说明

- **推论边界风险**：<第一条，加粗强调。①结论是否来自同系列推断；②目标产品与参考产品之间是否存在策略差异风险；③是否需人工向管理人确认。>
- 同系列产品证据可作支撑但置信度降低，需注明是否需人工确认。
- 字段缺失与策略定性冲突是两类问题，明确区分。
- 基准体系不一致时 fundType 不可直接复制，说明原因并标注人工确认点。
- <其他口径说明>
```

## Multiple Funds

综合报告使用列表格式。

```markdown
# <批次名>多基金策略标签审核报告

生成时间：YYYY-MM-DD HH:MM:SS / 审核范围：<用户输入> / 覆盖基金：<N> 只

## 结论摘要

- 一致：<N> 只
- 需补充/修正：<N> 只
- 人工复核：<N> 只

## 基金审核总览

- **<基金A>**：strategyType → <结论> / raceName → <结论> / fundType → <结论>
- **<基金B>**：...
```

## Brevity Rules

- 单基金：严格六段，不增不减
- 关键证据：3-6 条 + 证据链小结（多份且有时序演进时必写）
- 多基金：每行一只基金
- 不输出原始文件清单
- 表格单元格内换行：仅用 `<br>`，严禁物理回车
- 字段引用：全文用反引号包裹（如 `` `strategyType` ``）
- 收益来源：按子策略拆行，每行半句话以内

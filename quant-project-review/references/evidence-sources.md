# Evidence Sources

进行项目进度与偏航评估时，优先读取以下证据，而不是凭印象判断。

## 优先证据

1. 用户最初需求或任务说明
2. 计划文档、规格文档、任务拆解文档
3. `project.yaml`
4. 项目主目录结构
5. 运行入口、工作流入口、调试入口
6. 最近新增或修改的核心模块
7. 测试文件、验证脚本、回测结果、输出快照

## QuantSystem 中常见高价值信号

- 是否已有标准项目目录
- 是否已有 `src/`、`tests/`、`artifacts/`
- 是否已有稳定输出文件或快照
- 是否已有工作流接入
- 是否已有前端消费或统一输出契约
- 是否已有真实可复现入口，而不只是 Notebook 或临时脚本

## 低价值信号

以下内容不能单独说明项目进度高：

- 文件很多
- 代码行数很多
- 做了很多目录整理
- UI 页面很丰富
- 写了很多通用工具

只有当这些内容明确服务主链路时，才计入有效进展。

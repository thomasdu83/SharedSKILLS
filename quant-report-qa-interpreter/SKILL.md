# 定量报告质检与解读（Quant Report QA & Interpretation）

## 适用场景

当用户提供单一产品/管理人定量分析报告（通常为docx/markdown，包含指标、打分、场景因子等）并要求：
- 检查报告是否正常出具（空白、缺失、错误、不合理、口径混乱）
- 基于“评分用于初筛 + 有效因子用于二次解读 + 风控阈值用于一票否决/降权”的三段式框架做解读
- 针对不同策略类型（股票多头/市场中性/CTA）按对应因子的“建议定位”解释分数
- 场景因子结合当前所处场景做分析
- 输出Markdown/HTML解读报告并按规范命名文件

调用本技能。

## 输入（推荐）
- `report_path`：待解读定量报告路径（优先docx，其次markdown）
- `reference_dir`：参考材料目录（包含“因子有效性与评分结果解读/定量评价标准/因子有效性”文件）
- `policy_paths`：制度文件路径列表（用于投前风控与名单等级判定），至少包含：
  - `20260129 V2自主管理产品风险管理及流程办法【三次修订版】(1).docx`
  - `私募各级名单管理流程实施细则（修订）-2026年1月29日.docx`
- `asof`：报告截止日（若报告内未清晰给出，用文件名日期或文件修改日期替代）
- `strategy_type`：股票多头 / 市场中性 / CTA（如未给出，从报告标题/文件名/指标体系推断）
- `output_format`：`markdown` 或 `html`（默认html）
- `output_dir`：输出目录（默认与report_path同目录）
- `updated_by`：更新人（默认使用系统用户名或“AI”）

## 输出
- 生成1份解读报告文件（Markdown或HTML）
- 文件名必须包含：管理人名称、策略类型、日期（YYYYMMDD）
  - 示例：`创奥投资_CTA_定量报告解读_20260312.md`

## 执行步骤（必须严格执行）

### Step 0.5：开篇一段话结论（必须）
在展开任何章节细节之前，只输出**一段话**作为“开篇结论”，服务投前风控入池审批（定量侧），必须同时覆盖以下要素且保持精炼：
1. **总分结论**：总分/分位（如有）与“是否通过初筛”的一句话判定
2. **有效因子结论（A类）**：挑选1–3个最关键的A类因子/分项得分，说明其对“可持续性/Alpha”的指向
3. **描述/约束因子结论（B类）**：用1–2个B类指标说明“事故型/尾部风险约束”或“仅作描述”的原因
4. **近期场景匹配结论（C类）**：结合互联网信息用1-2句话说明当前所处的场景与该产品“场景应对能力/场景短板”的匹配度与影响
5. **入池与风险等级一句话**：仅基于制度门槛“定量得分>65 + 净值长度≥52周”判定是否符合入池标准

写作要求：
- 仅一段话，但需要注意分句，不用分点、不用小标题
- 必须可追溯：所有内容必须保证内容可追溯，或者有公开来源URL，附在最后
- 一段话中不要含有溯源的信息，如table、P、表、step等

### Step 0：读取与策略识别
1. 校验文件是否可读；优先从报告标题/封面/目录识别：
   - 管理人名称
   - 策略类型（股票多头/市场中性/CTA）
   - 报告区间与截止日
2. 若报告内缺失上述信息，使用文件名与路径字段推断，并在报告“口径与假设”中显式标注“推断来源与不确定性”。

### Step 1：报告质检（AI质检员）
对报告做“结构完整性 + 数据合理性 + 口径一致性”三类检查，输出“问题清单 + 修复建议”：

**A. 结构完整性**
- 目录/章节是否存在明显空白段落、占位符（如“XXX”“待补充”“Error/NaN”）
- 关键模块是否缺失（至少包含：产品概况、业绩/风险指标、打分或因子结果、场景分析/解释、结论建议）

**B. 数据合理性（举例）**
- 年化收益、波动、回撤、夏普/卡玛之间是否明显矛盾（如负收益+高夏普、回撤为正、波动为0等）
- 各分项得分是否越界（如<0或>满分）
- 同一指标在不同表/段落口径不一致（如回撤% vs 小数；年化% vs 小数）

**C. 口径一致性**
- 频率口径（周/月/日）是否混用且未说明
- 基准口径（对比基准/同类样本）是否一致
- 若报告是“预估/非托管口径”，必须显著提示

输出时以“可执行”的方式写：每条问题包括【位置标识（页/段落/行/表格）】【问题】【影响】【修复建议】。


### Step 2：三段式用法解读（核心框架）
必须按以下结构解释评分结果如何使用（不调整权重前提）：
1. **初筛（评分）**：用定量总分/分项分筛掉明显不合格者，但不把“高分”直接等同于“可投”。
2. **二次解读（有效因子）**：用“跨期更稳定的有效因子”解释为什么它值得更高/更低；识别高分来自“β”还是“α/残差”。
3. **一票否决/降权（风控阈值）**：把长期有效性弱但风险含义强的指标作为约束（阈值/预算/降权），避免“用风险指标去选α”。

### Step 2.5：入池标准与风险等级（仅得分 + 净值长度）
本步骤只做两件事：判断“是否满足入池标准”，并给出“风险等级（从低到高）”。除**得分**与**净值长度**外，不引入任何其他因素。

制度口径（名单细则docx段落号）：
- **净值长度达标**：净值长度≥52周（名单细则 P71、P80）
- **入池得分达标**：全区间归一化后的定量得分总分>65（名单细则 P74、P83）

执行规则（必须）：
1. 从报告中抽取“定量得分总分”的数值与口径说明；若报告未明确其为“全区间归一化定量总分”，标注“口径待确认”，但仍以报告给出的总分作为暂用值并提示需系统口径复核。
2. 从报告中抽取净值长度：
   - 优先使用报告明确写出的“净值长度（周）/起止日期”计算周数；
   - 若无法抽取，则判为“净值长度缺数据=不达标（投前不通过）”。
3. 入池标准判定：
   - 同时满足：得分>65 且 净值长度≥52周 → “通过”
   - 否则 → “不通过”（缺数据也视为不通过，并标注需补齐项）
4. 风险等级判定（仅基于得分与净值长度，低→高）：
   - 1级（低）：得分≥65 且 净值长度≥52周
   - 2级（中低）：得分≥65 且 （实盘净值长度≥39周，但小于52周、或者净值需进行拼接）
   - 3级（中）：得分<65 或者 净值长度<52周 且 无法提供足够托管净值
   - 4级（中高）：得分<65 或者 净值长度严重不足
   - 5级（高/不入池）：仅纯回测或模拟盘业绩，无实盘验证；无法提供任何有效的业绩替代材料

输出格式（必须，写在报告中对应章节）：
- 入池标准：通过/不通过
- 风险等级：1/2/3/4/5（低→高）



### Step 3：按策略类型解释“哪些指标该信、哪些指标只做描述”
结合参考材料中的因子有效性结论（以长期ICIR为主要依据），将指标划分为：
- **A类（选拔/加分型）**：跨期更稳定，可用于解释与排序
- **C类（场景型）**：短期有效，适合投后监控与当期环境匹配

#### 股票多头（参考：因子有效性 2026-03-12）
- A类（更值得信任）：`daywinrate`（ICIR≈0.61）、`annualyield`（≈0.39）、`sharpe`（≈0.29）、`alpha_industry`（≈0.30）、`alpha_style`（≈0.24）、`industryallocation`（≈0.20）
- B类（更偏描述/约束）：`annualdownsd`（≈-0.07）、`drawback`（≈-0.04）、`assetallocation`（≈-0.05）
- C类（场景）：如“市场下跌且波动上升”相关场景（长期ICIR偏弱但阶段性有效）

#### 市场中性（参考：因子有效性 2026-03-12）
- A类：`annualyield`（≈0.34）、`daywinrate/周胜率`（≈0.34）、`sharpe`（≈0.25）、`styleallocation`（≈0.22）、`industryallocation`（≈0.21）、`alpha_capacity`（≈0.16）
- B类：`annualdownsd`（≈-0.38）、`drawback`（≈-0.13）
- C类：与“市场上涨且波动上升”等场景相关指标（跨期不稳定，适合投后）

#### CTA（参考：因子有效性 2026-03-12）
- A类：`residual_race`（≈0.60）、`residual_style`（≈0.54）、`alpha_style`（≈0.32）、`annualyield`（≈0.21）
- B类：`annualdownsd`（≈-0.61）、`drawback`（≈-0.50）、`calmar`（≈-0.05）
- C类：传统/特殊/周期强弱/拥挤度等场景因子（部分子因子阶段性有效，必须结合当期场景解释）

### Step 4：场景因子分析（必须结合“当前所处场景判断”）
优先从报告内的“场景定义/场景收益表现/当期市场描述”提取“当前场景”。若报告缺失场景判断：
1. 使用 `asof` 作为查询窗口，结合公开互联网信息构造最小场景判断：
   - 股涨/股跌（例如中证800/沪深300）
   - 商涨/商跌（例如南华商品指数）
   - 波动上升/下降（例如波动率指数、或用收益率波动替代）
   - 相关性高/低（用阶段性相关性描述，不必精确数值）
2. 公开信息获取要求：
   - 必须使用公开来源（交易所/指数公司/主流财经媒体/券商研究公众号的公开稿等）进行交叉验证
   - 输出时必须提供“来源URL + 抓取日期 + 关键数值/结论摘录”，并标注用于判断的窗口（例如过去1周/1月）
3. 将场景映射到报告使用的场景体系（如：股涨商跌/股跌商涨/股商双牛/股商双杀；或“波动率上升/下降”等）。
4. 输出“场景结论 → 对该产品场景因子得分/表现的解释 → 投后建议（加仓/降配/观察/复调问题）”。

### Step 5：生成解读报告文件（Markdown/HTML）
输出至少包含以下章节：
1. 解读结论（放在最前，见 Step 0.5）
2. 摘要（3-6条）
3. 报告质检结论（问题清单 + 修复建议）
4. 评分结果与三段式用法（初筛/二次解读/一票否决）
5. 指标得分解读（按A/B/C类，结合策略类型）
6. 入池标准与风险等级（仅得分+净值长度，见 Step 2.5）
7. 场景因子与当期环境适配性（给出场景判断与含义 + 公开来源引用）
8. 附录：关键指标表（从报告抽取的主要数值与口径说明）

并将文件保存到 `output_dir`，文件名按规则生成。

## 质量门槛（必须满足）
- 所有“关键结论”必须对应至少一条可追溯证据（页码/段落/行/Slide/表格）
- 出现冲突信息必须标注“待确认”并列出需要补充的证据
- 不能凭空补全数据；只能做“合理性检查与疑点提示”
